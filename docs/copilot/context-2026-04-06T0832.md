## Phase 10 Deliverables

### 1. Rate Limiting (`src/lib/rate-limit.ts`)
- Upstash Redis sliding window algorithm
- **Fail-open mode:** Returns success when env vars missing (dev safety)
- `rateLimitByIp(request)` — 10 req/60s for export/pdf, 5 req/60s for auth
- `rateLimitByOrg(orgId)` — per-organization limits
- Returns: `{ success, remaining, reset?, retryAfter? }`

### 2. Storage Tracking (`src/lib/storage-tracker.ts`)
- Atomic upsert for `StorageUsage` model per org/category
- Categories: export, proof, attachment, pixel
- Functions: `trackStorageUsage`, `removeStorageUsage`, `getStorageUsage`, `getStorageSummary`

### 3. Job Recovery (`src/lib/job-recovery.ts`)
- Creates/completes/fails jobs with exponential backoff retry (2^retryCount * 30s)
- 3 max retries → dead letter queue on failure
- Functions: `createJob`, `markJobComplete`, `markJobFailed`, `getRetryableJobs`, `getDeadLetterJobs`, `retryJob`

### 4. Send Logger (`src/lib/send-logger.ts`)
- Fire-and-forget logging for all email operations
- Types: invoice_send, scheduled_send, reminder, notification
- Statuses: sent, failed, bounced
- Paginated retrieval: `getSendLogs(orgId, filters)`

### 5. Storage Adapter (`src/lib/storage-adapter.ts`)
- Interface-based abstraction: `StorageAdapter`
- **Supabase implementation** (current)
- **S3 stub** (ready for Phase 12)
- `getStorageAdapter()` reads `STORAGE_PROVIDER` env var

### 6. Sentry Integration (`src/lib/sentry.ts`)
- Thin wrapper: `captureError`, `captureMessage`, `setUserContext`
- Ready for full `@sentry/nextjs` integration when DSN configured

### 7. Middleware Updates (`src/middleware.ts`)
- Rate limiting on `/api/export/*`, `/api/pdf/*`, `/api/cron/*` routes
- Added public prefixes: `/share`, `/api/billing`
- Cron routes skip rate limiting (trusted)
- All existing auth logic preserved

### 8. Environment Variables (`src/lib/env.ts`)
Added optional vars:
```
UPSTASH_REDIS_REST_URL (optional)
UPSTASH_REDIS_REST_TOKEN (optional)
SENTRY_DSN (optional)
SENTRY_AUTH_TOKEN (optional)
STORAGE_PROVIDER (default: "supabase")
```

## Phase 11 Deliverables

### 1. Plan Infrastructure (`src/lib/plans/`)

**Config** (`config.ts`):
- 4 plans: Free / Starter ₹999 / Pro ₹2,999 / Enterprise ₹9,999
- `PlanLimits` interface: 16 limit fields + 6 boolean feature flags
- `PLAN_LIMITS` record with per-plan config
- `getPlan()`, `getPlanLimits()`, `formatPriceInr()` helpers

**Enforcement** (`enforcement.ts`):
- `getOrgPlan(orgId)` → reads `Subscription`, defaults to free
- `checkLimit(orgId, resource)` → checks usage vs plan limits, -1 = unlimited
- `checkFeature(orgId, feature)` → boolean feature flags
- `requirePlan(orgId, minimumPlan)` → throws if org plan below minimum
- `isTrialExpired(subscription)` → trial end date check

**Usage** (`usage.ts`):
- `incrementUsage(orgId, resource)` → atomic upsert to `UsageRecord` for current month
- `decrementUsage(orgId, resource)` → prevents going below 0
- `getMonthlyUsage(orgId, resource)` → returns current month count
- `getAllUsage(orgId)` → all resources for current month
- `resetMonthlyUsage(orgId)` → cleanup previous months

**Client Hook** (`src/hooks/use-plan.ts`):
- `usePlan(orgId)` — fetches plan info via `/api/billing/plan`
- Returns: `{ plan, loading }`
- Plan object: `{ planId, planName, status, limits, trialEndsAt, isTrialing, isFree }`

**API Endpoint** (`src/app/api/billing/plan/route.ts`):
- GET handler returning org plan info as JSON

**Components** (`src/components/plan/`):
- `UpgradeGate` — wrapper checking feature availability, shows lock + upgrade CTA if unavailable
- `UpgradePrompt` — modal for limit-reached with progress bar + plan comparison
- `UsageIndicator` — inline bar showing usage (green → yellow at 75% → red at 90%)

**API Routes** (`src/app/api/billing/razorpay/`):
