## Key Technical Decisions

- **AbortError handling**: `DOMException` with `name === "AbortError"` — workspace catches it and sets `status: "idle"` silently (not treated as an error state)
- **OCR cancellation**: `cancelAllOcr()` terminates the Tesseract.js worker. Workspace synchronously marks all `pending`/`processing` images as `cancelled`. State is distinct from `error`.
- **`cancelled` vs `error`**: `cancelled` images can be retried (user chose to stop). `error` images also retryable but represent genuine failures. Both show in retry list.
- **FormData transport**: Browser sets `multipart/form-data` boundary automatically — no manual `Content-Type` header. Server uses `request.formData()`.
- **Blob watermark persistence**: Blob URLs are ephemeral (session-scoped). On save, type is downgraded to `"none"` with `_hadImageWatermark = true`. On restore, a targeted message prompts re-upload.
- **Passwords never persisted**: `userPassword`, `confirmPassword`, `ownerPassword` always stripped to `""` / `undefined` before writing to localStorage.
