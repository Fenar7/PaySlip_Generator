### Key Fixes & Improvements (Recent)

1. **Template Rendering Bug (Fixed 2026-04-06)**
   - **Problem:** Document Canvas always showed a hardcoded layout, ignoring `templateId`
   - **Root Cause:** `InvoiceDocumentEditor`, `VoucherDocumentEditor`, `SalarySlipDocumentEditor` were static; didn't route to selected template
   - **Solution:** Replaced static editors with template preview components that route by `templateId`
   - **Commit:** `6d6d18d`

2. **selectedTemplateId Race Condition (Fixed 2026-04-06)**
   - **Problem:** Async `useWatch()` in `useState()` caused race; default template always selected even with URL param
   - **Solution:** `useState(() => getValues("templateId") ?? "professional")` — sync read from React Hook Form
   - **Commit:** `6d6d18d`

3. **Full Inline Editing for All 15 Templates (Fixed 2026-04-06, PR #48)**
   - **Problem:** After template fix, Document Canvas became read-only — no way to edit fields
   - **Solution:** Added `mode="edit"` to all templates, created `[Name]Editor` sub-components with inline fields
   - **Features:**
     - Text fields: inline edit with blur-to-save
     - Date fields: date picker
     - Number fields: numeric input
     - Line items: add/remove rows with `useFieldArray`
     - Totals/computed values: read-only, recalc on change
   - **Commit:** `a640838` (merged PR #48)
   - **Files:** 25 files modified, +3,780 lines

4. **Organization Creation (Fixed earlier phase)**
   - **Problem:** Profile upsert was failing silently
   - **Solution:** Added proper `UPSERT` logic in org creation action
   - **Status:** ✅ Working

## Architecture & Tech Stack

### Frontend Stack
- **Framework:** Next.js 15 (App Router)
- **Form Management:** React Hook Form + Zod validation
- **Styling:** Tailwind CSS + Shadcn/UI
- **State Management:** React Context (docs, auth), Server State (Next.js)
- **PDF:** PDFKit (Node.js server-side generation)
- **Image Processing:** Sharp

### Backend Stack
- **Runtime:** Node.js (Vercel)
- **Database:** PostgreSQL (Supabase)
- **ORM:** Prisma 7 (client provider pattern)
- **Auth:** Better Auth (JWT + refresh tokens)
- **Jobs:** Trigger.dev (recurring, delayed)
- **Email:** Resend
- **File Storage:** Supabase Storage
- **Monitoring:** Sentry (ready, not yet wired)

### Key Patterns & Conventions

**Prisma 7:**
- Import: `import { PrismaClient } from "@/generated/prisma/client"`
- Config: Driver adapter pattern in `src/lib/db.ts`
- Nullable JSON: Use `Prisma.DbNull` (not `null`), cast to `Prisma.InputJsonValue`

**Form & Canvas Sync:**
- Templates receive `Document` prop + form `values` via `useFormContext()`
- In `mode="edit"`: template calls `useFieldArray()` and `useWatch()` to edit form fields
- In `mode="preview"`: template is read-only display
- Changes in form auto-sync to canvas; changes in canvas auto-sync to form

**Inline Editing:**
- All editable fields use `InlineTextField`, `InlineTextArea`, `InlineDateField`, `InlineNumberField`
- These components call `useFormContext()` internally
- Computed fields (totals, tax, subtotals) remain read-only

**ActionResult Pattern:**
```typescript
type ActionResult<T> = 
  | { success: true; data: T } 
  | { success: false; error: string };
```
Defined per-file, not shared.

**RBAC:**
- 7 roles: Owner → Approver → Editor → Member → Guest → Viewer
- 15 modules: docs.invoices, docs.vouchers, docs.salary_slips, pdf_studio, pixel, pay, flow, intel, settings, users, roles, audit, exports, admin, api
- 7 actions: create, read, update, delete, approve, export, share
- Enforcement: `requirePermission(orgId, userId, module, action)` in server actions
- Proxy grants for cross-org impersonation with full audit trail

### Phase 11 — Subscription Billing + Commercial Launch + Growth (3 sprints)

**🇮🇳 Payment Gateway: Razorpay (India-first) — NOT Stripe**
- **Why Razorpay?** 1–2 week approval vs. Stripe's 1–3 months, UPI support (60% of Indian users), 0% UPI fees, GST-ready
- **Future:** Stripe can be added in Phase 12+ for global expansion

**Sprint 11.1 — Pricing Model & Plan Infrastructure:**
- 4 tiers: Free / Starter ₹999/mo / Pro ₹2,999/mo / Enterprise custom
- 14-day Pro trial for all new orgs
- Feature gating in server actions (not just UI) to prevent API bypass
- `checkLimit()` + `incrementUsage()` enforcement in every critical action
- Feature matrix: 20+ features × 4 plans

**Sprint 11.2 — Razorpay Billing Integration:**
- Full subscription lifecycle via Razorpay plans (auto-recurring)
- Payment methods: UPI (mandate), Cards, Netbanking
- Razorpay webhooks: subscription.authenticated, payment.failed, invoice.issued
- Trial → upgrade flow (skips remaining trial when user subscribes)
- Idempotent webhook processing via `RazorpayEvent` table

**Sprint 11.3 — Growth & Conversion:**
- Public marketing pages: `/pricing`, `/pdf-studio`, `/invoice-generator`, etc.
- Anonymous → registered funnel: 3 free PDF Studio uses without signup
- Referral system with credit rewards
- Onboarding checklist for new orgs
- Shareable invoice links (public, no auth required)
