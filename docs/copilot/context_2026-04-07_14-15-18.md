## Key Technical Decisions

### Ledger-Driven Invoice Reconciliation
`src/lib/invoice-reconciliation.ts` is the **single canonical source of truth** for invoice status:
- All payment paths (`markInvoicePaid`, `recordPayment`, `acceptProof`, `rejectProof`, webhook handlers) call `reconcileInvoicePayment()`
- Status is **derived** (not directly set): sums SETTLED ledger rows → computes amountPaid/remainingAmount → determines status atomically
- Idempotent: `InvoiceStateEvent` only created when status actually changes

### InvoicePayment Status Enum
- `SETTLED` — counts toward paid totals
- `PENDING_REVIEW` — proof uploaded, not yet reviewed
- `REJECTED` — proof rejected
- `OVERPAID_REVIEW` — gateway overpayment
- `FAILED` — payment failed

### Idempotency Pattern
Webhook handlers guard on `externalPaymentId`:
```typescript
const existing = await db.invoicePayment.findUnique({ 
  where: { externalPaymentId: razorpayPaymentId } 
});
if (existing) return; // Already processed
```
Prevents duplicate Razorpay webhooks from double-counting payments.

### Proof-to-Ledger Linking
`InvoiceProof.invoicePaymentId` (unique FK) links each proof to its ledger row:
- Accepting proof → settles linked ledger row → reconciles
- Rejecting proof → marks ledger REJECTED → reconciles (never blindly resets to ISSUED)

## Database Schema Highlights

### Invoice Model (Reconciliation Fields)
```prisma
model Invoice {
  // ... existing fields ...
  
  // Reconciliation snapshots (derived by reconcileInvoicePayment)
  amountPaid           Float           @default(0)
  remainingAmount      Float           @default(0)
  lastPaymentAt        DateTime?
  lastPaymentMethod    String?         // admin_manual, razorpay_payment_link, smart_collect, etc.
  paymentPromiseDate   DateTime?       // promised delivery date
  paymentLinkStatus    String?         // active, expired, completed
  paymentLinkLastEventAt DateTime?
  
  // Relations
  payments             InvoicePayment[]
  proofs               InvoiceProof[]
  stateEvents          InvoiceStateEvent[]
}

model InvoicePayment {
  id                  String      @id @default(cuid())
  invoiceId           String
  invoice             Invoice     @relation(fields: [invoiceId], references: [id], onDelete: Cascade)
  
  // Reconciliation fields
  source              String      // admin_manual, public_proof, razorpay_payment_link, smart_collect, api
  status              String      @default("SETTLED") // SETTLED, PENDING_REVIEW, REJECTED, OVERPAID_REVIEW, FAILED
  amount              Float
  externalPaymentId   String?     @unique @index // Razorpay payment ID (idempotency key)
  transactionRef      String?     // bank txn ref
  bankName            String?
  failureReason       String?
  
  // Audit
  recordedBy          String
  recordedAt          DateTime    @default(now())
  reviewedBy          String?
  reviewedAt          DateTime?
  
  // Relations
  proof               InvoiceProof?
  
  @@index([invoiceId])
  @@index([externalPaymentId])
}

model InvoiceProof {
  id                  String      @id @default(cuid())
  invoiceId           String
  invoice             Invoice     @relation(fields: [invoiceId], references: [id], onDelete: Cascade)
  
  invoicePaymentId    String      @unique
  payment             InvoicePayment @relation(fields: [invoicePaymentId], references: [id])
  
  fileName            String
  fileUrl             String
  fileType            String
  fileSizeBytes       Int
  uploadedAt          DateTime    @default(now())
  uploadedBy          String
  
  // For partial payments
  plannedNextPaymentDate DateTime?
  
  @@index([invoiceId])
}
```

- Zod schemas for all inputs (invoices, vouchers, salary slips, payments)
