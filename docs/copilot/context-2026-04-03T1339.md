## Architecture Summary

### Feature: Direct Edit View
Both Form and Document views share the same `react-hook-form` context via `FormProvider` — zero sync needed.

**Toggle:** `DocumentWorkspaceLayout` has `viewMode: "form" | "document"` state. When "document", form + preview panels are hidden; `documentEditorContent` renders full-width.

**Template → Editor mapping:**
- Invoice editor → mirrors `src/features/invoice/templates/minimal.tsx`
- Voucher editor → mirrors `src/features/voucher/templates/minimal-office.tsx`
- Salary Slip editor → mirrors `src/features/salary-slip/templates/modern-premium.tsx`

**Design tokens used in editors:**
- `--voucher-ink: #1d1710` — primary text
- `--voucher-accent: branding.accentColor || "var(--accent)"` — accent color
- Labels: `text-[0.68rem] font-semibold uppercase tracking-[0.25em] text-[rgba(29,23,16,0.45)]`
- Cards: `rounded-[1.5rem] border border-[rgba(29,23,16,0.08)]`
- Accent boxes: `rounded-[1.5rem] p-5 text-white` with `backgroundColor: "var(--voucher-accent)"`

**InlineEditFields (shared primitives):**
- `InlineTextField`, `InlineTextArea`, `InlineNumberField`, `InlineDateField`, `InlineSelectField`
- All use `text-[var(--voucher-ink)]`, `border-b-[var(--voucher-accent)]` on focus
- `DocumentEditorRoot` wrapper injects CSS vars and sets A4 max-width

**Line items:** `useFieldArray` with add/remove row buttons. Totals computed via `useWatch` + normalize functions.
