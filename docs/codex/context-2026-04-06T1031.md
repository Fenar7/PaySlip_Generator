## Completed Work

### 1. Org Creation Bug — FIXED ✅
**Symptom:** "Could not create organization. Try a different name." for any org name  
**Root cause:** `createOrg()` tried to create a `Member` record with FK to `Profile`, but new Supabase auth users had no `Profile` row yet → FK constraint violation  
**Fix:**
- `src/app/app/actions/org-actions.ts` — Added `db.profile.upsert()` before `db.member.create()`
- `src/app/onboarding/page.tsx` — Passes `userEmail` + `userName` to `createOrg()`; improved error messages to distinguish slug conflicts from general failures
- Committed directly to master: `fix(onboarding): upsert Profile before creating org member record`

### 2. Template Rendering Bug — FIXED ✅
**Symptom:** Document Canvas showed wrong/hardcoded layout regardless of which template was selected  
**Root cause:** All three `XxxDocumentEditor` components were hardcoded layouts ignoring `templateId` entirely  
**Fix:**
- Replaced `documentEditorContent={<InvoiceDocumentEditor />}` with `documentEditorContent={<InvoicePreview document={previewDocument} />}` in all three workspaces

## Key Technical Decisions Made

### Template Edit Mode Pattern
- `mode` type union extended to include `"edit"` across all 15 templates
- `data-testid="invoice-render-ready"` only set for `print/pdf/png` modes (not `preview` or `edit`)
- Template registry `index.tsx` types updated to include `"edit"` in component signature
- No changes to `preview/print/pdf/png` code paths — export functionality fully preserved

### selectedTemplateId Initialization Fix
- **Problem:** `useState(values.templateId)` where `values` comes from `useWatch()` → race condition on first render → always initialized to `"professional"` (form default) even when `?template=classic-bordered` was in URL
- **Fix:** `useState(() => getValues("templateId") ?? "professional")` — `getValues()` is synchronous and reads the initial `defaultValues` set from URL param

## Architecture Reference

### Template Flow (fixed)
1. `?template=classic-bordered` → `NewInvoicePage` reads `searchParams.template`
2. Passes `initialTemplateId` to `InvoiceWorkspace`
3. Sets `defaultValues.templateId = "classic-bordered"` in `useForm`
4. `InvoicePanel` reads `getValues("templateId")` sync → `selectedTemplateId = "classic-bordered"`
5. `previewDocument = normalizeInvoice({ ...values, templateId: selectedTemplateId })`
6. Form view: `InvoicePreview` → `InvoiceDocumentFrame mode="preview"` → correct template ✅
7. Document Canvas: `InvoiceEditableCanvas` → `InvoiceDocumentFrame mode="edit"` → `ClassicBorderedEditor` ✅

