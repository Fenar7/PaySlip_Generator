## 5. Architecture Rules to Always Respect

These are non-negotiable patterns established across Phases 16–17:

1. **Transaction safety:** Never call `fireWorkflowTrigger` inside a `db.$transaction(async tx => { ... })` callback. Always `await` the transaction first, then fire the trigger.
2. **No `revalidatePath` in library files:** `revalidatePath` is only valid inside Server Actions (`"use server"` files in `/app/`). Pure lib files under `src/lib/` must never call it.
3. **Auth pattern:** Always `await requireOrgContext()` at the top of every server action or API route. Never trust orgId from client input.
4. **Result shape:** Server actions follow `{ success: true, data } | { success: false, error: string }`. API routes return `NextResponse.json({ success: true, ...data })`.
5. **Cron auth:** Every cron route must call `validateCronSecret(request)` and return 401 if it fails.
6. **Audit completeness:** Every Flow action must emit to `ActivityLog` or `AuditLog`. Workflow engine writes `WorkflowRun` + `WorkflowStepRun` records automatically.
7. **Idempotency:** All cron jobs check terminal state before acting. Dead-letter jobs are idempotent (check `resolvedAt` before re-notifying).
