## Most Recent Issue Fixed

### Gemini GitHub Actions PR failure

The PR was failing because repo-owned Gemini workflows were automatically running without any configured Gemini authentication.

Observed root cause:

- `.github/workflows/gemini-dispatch.yml` triggers Gemini workflows on PR open / issue events / comments
- downstream workflows use `google-github-actions/run-gemini-cli@v0`
- no usable auth was configured via:
  - `secrets.GEMINI_API_KEY`
  - `secrets.GOOGLE_API_KEY`
  - or GCP WIF variables
- Gemini CLI exited with auth error, and the workflow then failed

### Fix applied

Added auth guards to these workflows so they self-skip when Gemini auth is not configured:

- `.github/workflows/gemini-review.yml`
- `.github/workflows/gemini-invoke.yml`
- `.github/workflows/gemini-plan-execute.yml`
- `.github/workflows/gemini-triage.yml`
- `.github/workflows/gemini-scheduled-triage.yml`

Behavior now:

- if Gemini auth exists, workflows still run normally
- if Gemini auth does not exist, Gemini steps are skipped with a clear log message instead of failing the PR
