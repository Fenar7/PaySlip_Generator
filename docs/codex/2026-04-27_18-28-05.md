## Important Runtime Findings During Debugging
- The repeated MFA loop was not one single bug. It came from several issues across iterations:
  - incorrect MFA requirement semantics
  - inconsistent token vs cookie handoff approaches
  - edge/runtime unsafe verification helpers
  - stale callback URLs carrying old `mfaToken` values
  - wrong `verifyOtp` parameter (`token` vs `token_hash`) in primary passkey sign-in
- The `Sign in` topbar bug on authenticated app pages was caused by the topbar relying only on the client auth hook even though the app shell had already been server-authenticated.
