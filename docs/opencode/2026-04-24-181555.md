## Architecture Decisions

- **OCR workspace uses separate localStorage key prefix** (`pdf-studio-ocr-session-v1`) rather than overloading `PdfStudioSession` — avoids type mismatch since OCR workspace doesn't use `PageSettings`
- **Only `complete` OCR state is persisted** — failed/pending/cancelled are intentionally dropped because they require source files to re-run
- **Data URLs for previewUrl are serializable and restorable** — `loadImageFromFile` uses `FileReader.readAsDataURL`
- **`generatePdfFromImages` already compatible** — consumes `item.ocrText` when `settings.enableOcr` is true, works with restored data

---

## Known Limitations / Notes

- OCR workspace session does **not** store `sourceFile` (File is not serializable)
- Users must re-upload source file to re-run OCR, but restored text is still exportable
- `poolMatchGlobs` + `forks.singleFork` in vitest config slows full suite slightly (~70s vs ~30s) but ensures correctness
